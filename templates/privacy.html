Privacy Policy

Fabric Trend Spotter ("we", "our", or "the app") is a fabric trend analysis tool that aggregates publicly available data from online marketplaces and social media platforms. This privacy policy explains how we collect, use, and protect information.

1. Information We Collect

We collect and process the following types of information:

  • Publicly Available Marketplace Data: Product listings, prices, ratings, and tags from public fabric marketplace pages (Etsy, Amazon, Spoonflower, and European fabric retailers). No user account data or private seller information is collected.
  • Google Trends Data: Aggregate search interest data for fabric-related keywords via the Google Trends API. This data is anonymous and aggregated by Google.
  • Instagram Hashtag Data: Public post counts and recent media metadata associated with fabric-related hashtags via the Instagram Graph API. We do not collect any private user data, direct messages, personal profiles, or non-public content.
  • Usage Analytics: Basic, anonymous usage data such as page views and feature usage to improve the application. No personally identifiable information is collected.

2. How We Use Information

All collected data is used exclusively for:

  • Analyzing and displaying fabric trend data on our dashboard
  • Generating trend forecasts and market insights
  • Comparing trends across different markets, countries, and regions
  • Tracking hashtag popularity for fabric-related social media trends

3. Instagram Data Usage

Our use of the Instagram Graph API is strictly limited to reading public hashtag data. Specifically, we:

  • Search for public hashtags related to fabric, sewing, and textile trends
  • Retrieve recent media metadata (post count, media type, timestamps) for trend analysis
  • Do not access private user profiles, direct messages, or personal data
  • Do not post, comment, or modify any content on Instagram
  • Do not store Instagram user IDs or personal information
  • Do not sell, share, or transfer Instagram data to third parties

Instagram data is cached temporarily (up to 24 hours) to respect API rate limits and is refreshed periodically to maintain accurate trend information.

4. Data Storage and Security

  • All data is stored in a server-side SQLite database with WAL mode enabled
  • No personal user data or credentials are stored
  • The database contains only aggregated marketplace and trend data
  • API keys and tokens are stored as environment variables, never in source code

5. Third-Party Services

We integrate with the following third-party services:

  • Google Trends (via pytrends): For search trend data
  • Instagram Graph API (Meta): For hashtag trend data
  • Chart.js: For rendering charts (loaded from CDN)

Each third-party service has its own privacy policy. We recommend reviewing their policies for information about how they handle data.

6. Data Retention

Trend data is retained to enable historical analysis and forecasting. Marketplace listing data is refreshed regularly. Instagram hashtag data is cached for up to 24 hours before being refreshed. No personal user data is retained as none is collected.

7. User Rights

Since we do not collect personal data from users, there is no personal data to access, modify, or delete. If you believe any publicly available data about your marketplace listings should be removed, please contact us.

8. Data Deletion

Users can request deletion of any data by contacting us. Upon receiving a valid deletion request, we will remove the specified data within 30 days. To request data deletion for Instagram-related data, we will comply with Meta's data deletion requirements.

9. Children's Privacy

This application is not directed at children under 13. We do not knowingly collect any information from children.

10. Changes to This Policy

We may update this privacy policy from time to time. Changes will be reflected by updating the "Last updated" date at the top of this page.

11. Contact Us

If you have questions about this privacy policy or our data practices, please reach out via our GitHub repository .
